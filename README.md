# ozteamproject
오즈코딩스쿨 1팀(민규, 세진, 지호, 영택, 보현)

이번 프로젝트가 '단순 제출용으로 만들어졌다가 버려지는 과제'가 아니라, 우리 모두의 이력서와 포트폴리오에 진짜 강력하게 남는 실전 자산이 되길 바랍니다.
이를 위해 앞으로 우리가 아이디어를 내고 선택할 때, 가장 핵심이 되는 기준을 정해두고 진행하고자 합니다. 어떤 아이디어든 아래 원칙을 중심으로 함께 검토해보면 좋겠습니다!


1. 쉬운 구조, 그러나 높은 유저 경험(UX) 완성도

복잡하고 어설픈 비즈니스 모델을 흉내 내기보다, 누구나 1초 만에 이해할 수 있는 직관적이고 슬림한 서비스를 지향합니다.
기능을 무리하게 늘리기보다, 실제로 유저가 들어와서 매끄럽게 사용할 수 있는 완성도 높은 MVP(최소 기능 제품) 구현에 집중합니다.

2. 전원 크로스펑셔널(Cross-functional) 경험

역할이 고정된 프로젝트가 아니라, 개발 · 마케팅(인스타/쓰레드) · 운영 전 과정을 팀원 모두가 주기적으로 순환(Rotation)하며 직접 겪어봅니다.
"서비스의 기획-개발-마케팅-운영 전체 파이프라인을 다 다뤄봤다"는 드문 포트폴리오 스토리를 목표로 합니다.

3. 명확하고 증명 가능한 실행 지표(KPI) 확보

실제 데이터, 콘텐츠 생성 및 전환율, SNS 도달률 등 서비스 운영을 통해 실제 측정되는 지표를 쌓습니다.
이 지표와 데이터가 향후 포트폴리오가 됩니다.


위 기준을 염두에 두시고 "이런 아이디어라면 위 기준을 다 만족하면서 재미있게 해볼 수 있겠다!" 싶은 생각이 있다면 자유롭게 던져주세요.
