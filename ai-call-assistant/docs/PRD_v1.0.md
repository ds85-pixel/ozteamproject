# AI 전문직 전화 비서 서비스 기획안 (v1.0)

## 1. 프로젝트 정의

세무사·변호사 등 전문직이 부재 중이거나 상담 중일 때, 전화를 놓치지 않고 **AI 음성 비서가 대신 응대하여 고객의 용건을 수집·요약해 대시보드로 전달하는 B2B SaaS 프로덕트**다.

전화 응대를 AI가 전처리(Pre-processing)함으로써, 전문직 종사자가 상담 준비 시간을 80% 이상 단축하고 핵심 업무에만 집중할 수 있는 환경을 제공한다.

---

## 2. 기획 배경

1. **상담 전화의 병목 현상:** 세무사 등 전문직은 하루 수십 통의 문의 전화를 받는다. 그러나 이동, 미팅, 서류 작성 중에는 전화를 받을 수 없어 부재중 전화가 누적된다.
2. **비효율적인 콜백(Callback) 프로세스:**
   * 기존: 부재중 확인 → 전화 걸기 → 용건 청취 → "확인 후 다시 연락드리겠다" 안내 → 세무 조사/확인 → 재연락 (총 5단계)
   * 문제점: 단순 용건 확인에만 과도한 시간 소요, 메모 누락, 고객 유실 발생.
3. **기존 전화 대행의 한계:** 일반 ARS는 기계적이라 고객 이탈률이 높고, 사람 비서 대행 서비스는 비용이 비싸며 세무 전문 용어 수집에 한계가 있다.

> **Our Solution:** AI가 사람처럼 자연스럽게 통화하여 핵심 용건(이름, 연락처, 세무 질문 내용)을 녹음·요약하고, 세무사는 **"요약된 리포트를 읽고 곧바로 정답을 들고 콜백"**하는 시스템을 구현한다.

---

## 3. 서비스 구조: 전화 응대 엔진 & 세무사 대시보드

서비스는 **실시간 AI 응대 엔진(전화망)**과 **세무사 전용 대시보드(웹/앱)** 두 축으로 구성된다.

```
[고객 전화] ──(부재/통화중 착신전환)──> [AI 전화 비서 (VoIP/STT/LLM)]
                                              │
                                    (용건 요약 & 카테고리화)
                                              │
                                              ▼
[세무사 콜백 & 업무 완료] <──(카톡/웹 알림)─── [세무사 전용 대시보드]
```

* **전화 응대 축:** 전화 수신 → AI 자연어 음성 응대 → STT 변환 → LLM 용건 요약
* **관리자 축:** 실시간 용건 알림(카카오톡/Push) → 요약 카드 확인 → 1-Click 콜백 → 상담 이력 관리

---

## 4. 핵심 사용자

1. **세무사 (Primary User)**
   * 부재중 전화로 인한 고객 유실을 막고 싶은 개업 세무사.
   * 콜백 전 미리 용건을 파악하여 상담 효율을 극대화하고 싶은 전문가.
2. **문의 고객 (End User)**
   * 전화를 받지 않아 무작정 기다리는 대신, 자신의 용건을 빠르게 전달하고 빠른 답변을 원하는 고객.

---

## 5. 핵심 기능 상세

### 5-1. AI 전화 응대 (Voice Agent)

* 세무사가 전화를 받을 수 없을 때 **'부재중 착신전환'**을 통해 AI 비서 번호로 연결.
* 사람과 대화하듯 자연스러운 음성(TTS)으로 인사 및 용건 수집.
* *응대 스크립트 예시:*
  > "안녕하세요, OO 세무사 사무실 AI 비서입니다. 세무사님이 현재 상담 중이셔서 메시지를 남겨주시면 바로 전달해 드리겠습니다. **어떤 세무 관련 건으로 전화주셨나요?**"

### 5-2. AI 용건 요약 파이프라인

* **STT (Speech-to-Text):** 음성을 텍스트로 변환 (세무 전문 용어 딕셔너리 적용).
* **LLM 요약:** 텍스트 데이터에서 핵심 정보 추출 및 구조화.
  * Extracted Data: `고객명`, `연락처`, `세목 카테고리(양도세/증여세/부가가치세 등)`, `주요 문의 요약(3줄)`, `긴급도(상/중/하)`

### 5-3. 세무사 대시보드 (Web/Mobile)

* **실시간 리드 카탈로그:** 들어온 부재중 용건을 카드 형태로 시각화.
* **1-Click 콜백:** 대시보드에서 전화 버튼 클릭 시 고객에게 즉시 연결.
* **상담 메모 및 상태 변경:** `대기 중`, `콜백 완료`, `상담 완료` 상태 관리.

## 6. 경쟁 서비스와의 차별점

| 구분 | 일반 전화 ARS | 사람 비서 대행 | 본 서비스 (AI 전화 비서) |
| :--- | :--- | :--- | :--- |
| **비용** | 매우 저렴 | 높음 (월 수십만 원~) | **합리적 (월 구독형 SaaS)** |
| **응대 퀄리티** | 딱딱함 (버튼 입력) | 높음 | **자연스러운 음성 대화 (AI)** |
| **전문성/요약** | 불가능 | 일반 메시지 전달 수준 | **세무 전문 용어 인지 & 3줄 요약** |
| **처리 속도** | 즉시 | 대행업체 전달 시간 소요 | **전화 종료 후 5초 내 알림 전달** |

## 7. MVP 개발 범위

3주~4주 개발 기간을 고려한 범위 설정.

### 반드시 만든다 (MVP)

* 통신사 착신전환 수신용 VoIP 전용 번호 연동 (Twilio / WebRTC)
* AI 음성 응대 및 대화 로직 (OpenAI Realtime API / Whisper STT + GPT-4o + TTS)
* 세무 전문 용어 추출 프롬프트 파이프라인
* 세무사 웹 대시보드 (용건 요약 리스트 조회, 상세 보기, 콜백 상태 업데이트)
* 카카오 알림톡 / SMS 즉시 알림 연동

### 다음 단계로 미룬다 (v2.0)

* 세무사 개인 음성 클로닝 (세무사 본인 목소리로 AI 응대)
* 홈텍스 / 세무 프로그램(더존 등) 데이터 자동 연동
* 챗봇 기반 사전 서류 접수 기능

---

## 8. 기술 스택 (Recommended Tech Stack)

* **Telephony / Voice:** Twilio (VoIP), OpenAI Realtime API / Clova Speech
* **Backend:** Node.js (NestJS) 또는 Python (FastAPI)
* **Frontend:** React / Next.js, Tailwind CSS
* **Database:** PostgreSQL / Supabase
* **Infrastructure:** AWS / Vercel

---

## 9. 핵심 성과지표 (KPI)

1. **AI 응대 완료율:** 전화 이탈 없이 용건을 끝까지 남긴 비율 (Target: > 70%)
2. **콜백 준비 시간 단축률:** 세무사가 콜백 전 사전 조사에 소요하는 시간 변화 (Target: 50% 이상 단축)
3. **요약 정확도:** AI가 요약한 내용과 실제 고객 용건의 일치율 (Target: > 90%)
