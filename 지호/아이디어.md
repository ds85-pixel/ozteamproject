# 1인 크리에이터를 위한 AI 자동화 레시피 플랫폼

**오즈코딩스쿨 1팀** · 민규 · 세진 · 지호 · 영택 · 보현

## 1. 문제 정의와 한 줄 요약 (Problem & Elevator Pitch)

> **코딩을 모르는 1인 유튜버가, AI 자동화를 '내 채널'에 바로 적용하게 해주는 서비스** *(47자)*

### 문제의 근거 (객관적 사실 기반)

AI 자동화 정보는 이미 넘쳐난다. 인프런의 클로드 코드 강의는 수강생 13,000명 이상, 지피터스·한국GPT협회 같은 커뮤니티, n8n 오픈소스 워크플로우 4,300개+ 가 존재한다. 즉 **'방법'은 부족하지 않다.**

문제는 코딩을 모르는 1인 크리에이터가 이 정보를 **자기 채널에 실제로 적용하는 지점**에서 발생한다. 조사 결과 세 가지 벽이 반복적으로 확인된다.

1. **정보는 많은데 못 쓴다** — 방법은 넘치지만, 내 상황(채널 주제·OS·사용 툴)에 맞게 고치는 지점에서 막힌다.
2. **툴이 매주 바뀐다** — "업데이트 쫓기 피곤하다"는 반응이 잦다. 어제 배운 방법이 오늘 안 되는 경우가 많다.
3. **에러 앞에서 멈춘다** — 따라 하다 에러가 나면 혼자 해결하지 못하고 자동화 자체를 포기한다.

기존 자료는 전부 '방법을 설명'하는 데 그친다. 이 서비스는 '내 것으로 바꿔주고, 끝까지 완성하게 돕는' 데 초점을 맞춘다.

## 2. 핵심 비즈니스 로직 (Core Business Logic)

### 유저 여정 (진입 → 목적 달성 → 이탈)

| 단계 | 유저 행동 | 서비스 동작 |
|---|---|---|
| ① 진입 | "썸네일 만드는 거 자동화하고 싶다" | 카테고리별 레시피 목록 노출 (썸네일/자막/업로드/설명 작성) |
| ② 선택 | 원하는 레시피 클릭 | 레시피 상세 + '내 상황 입력' 폼 표시 |
| ③ 입력 | 채널 주제·OS·사용 툴 입력 | **AI가 레시피를 그 조건에 맞게 자동 변형** |
| ④ 실행 | 변형된 결과 복사 → 자기 환경에 붙여넣기 | 복사 버튼 + 단계별 실행 가이드 제공 |
| ⑤ (막힘) | 에러 발생 시 '막혔어요'에 에러 붙여넣기 | 그 레시피 맥락에서 해결책 제시 |
| ⑥ 목적 달성 | 자동화 작동 확인 후 이탈 | 완료 → 관련 레시피 추천 / 공유 유도 |

### 타겟 유저가 '굳이' 이 서비스를 써야 하는 이유

- **강의 대비:** 1~2시간짜리 영상을 다 볼 필요 없이, 지금 필요한 레시피 하나만 복붙한다. 시간이 절대 부족한 1인 크리에이터에게 결정적이다.
- **커뮤니티 대비:** 질문하고 답변을 기다릴 필요 없이, 내 상황에 맞게 변형된 결과를 즉시 받는다.
- **범용 AI 툴(챗GPT 등) 대비:** "어떻게 물어봐야 하는지"부터 막히는 초보에게, 검증된 레시피가 프롬프트 설계를 대신해준다.
- **결정적 차이:** 기존 어떤 서비스도 '내 채널 조건에 맞춘 변형'과 '에러 해결까지'를 함께 제공하지 않는다.

## 3. 최소 기능 구현 명세 (MVP Features)

'있으면 좋은 기능'은 전부 배제하고, 서비스가 굴러가기 위한 핵심 기능만 도출한다.

### 필수 기능 (이게 없으면 서비스가 성립 안 됨)

**F1. 레시피 목록 & 상세 페이지**
- 크리에이터용 레시피 10~15개를 카테고리(썸네일/자막/업로드/설명)로 분류해 노출.
- 각 레시피는 '무엇을 자동화하는지 / 필요한 툴 / 복붙할 내용'으로 구성.

**F2. 내 상황 입력 → 자동 변형 (핵심 차별점 ①)**
- 유저가 채널 주제·OS·사용 툴을 폼에 입력.
- 이 값을 원본 레시피와 함께 Claude API에 전달 → 조건에 맞게 재작성된 프롬프트/설정을 반환.
- 예: 원본 "썸네일 자동 생성 레시피" + 입력 "윈도우, 사주 채널" → 윈도우 경로·사주 톤에 맞춰 변형된 결과 출력.

**F3. 복사 버튼 & 실행 가이드**
- 변형된 결과를 원클릭 복사.
- 붙여넣어 실행하는 순서를 번호 단계로 함께 표시.

**F4. '막혔어요' 에러 도우미 (핵심 차별점 ②)**
- 유저가 에러 메시지를 붙여넣으면, 해당 레시피 맥락 + 에러를 Claude API에 전달 → 원인과 해결책 반환.

### 다음 단계로 미룸 (MVP 제외)
사용자 레시피 업로드·심사 / 레시피 자동 최신화 / 결제·프리미엄 / 다른 니치 확장 / 로그인·회원 시스템(초기엔 비로그인으로도 동작 가능)

## 4. 기술 스택 및 데이터 확보 방안 (Tech Stack & Feasibility)

### 기술 스택 (확정)

| 구분 | 기술 | 선택 이유 |
|---|---|---|
| 프론트엔드 | React + Vite | 팀 부트캠프 커리큘럼과 일치, 빠른 MVP 구현 |
| 스타일 | Tailwind CSS | 슬림한 UI를 빠르게 완성 |
| 백엔드 | Supabase (Edge Functions) | 별도 서버 구축 없이 API 중계·DB·인증을 한 번에 |
| DB | Supabase(PostgreSQL) | 레시피 데이터·변형 로그·KPI 집계 저장 |
| AI 엔진 | Claude API | '자동 변형'과 '에러 도우미'의 핵심. 아래 비용 근거 참조 |
| 배포 | Vercel | 프론트 무료 배포, 깃헙 연동 자동 배포 |

### 외부 API 데이터 확보 가능 여부 및 제약 (실측 확인)

**① Claude API — 확보 가능, 비용 관리 가능**
- 2026년 7월 기준 Haiku 4.5는 100만 토큰당 입력 $1 / 출력 $5, Sonnet 5는 입력 $2 / 출력 $10(8월 31일까지 인트로 가격). *(출처: Anthropic 공식 pricing)*
- 레시피 변형 1건은 입출력 합쳐 수천 토큰 수준 → **건당 1~2원 내외.** 단순 변형은 저가 모델(Haiku)로 라우팅, 프롬프트 캐싱(캐시 입력 90% 절감)으로 비용을 더 낮춘다.
- **결론:** MVP 트래픽 규모에서 비용 부담 없이 운영 가능.

**② YouTube Data API v3 — 제약 존재, 그러나 우리 구조에선 병목 아님 (중요)**
- YouTube Data API는 프로젝트당 **하루 10,000 유닛** 무료. 영상 업로드는 약 1,600유닛(2025년 12월 인하 후 약 100유닛 + 2026년 6월부터 업로드 전용 버킷으로 분리). 검색은 1회 100유닛으로 비싼 편. *(출처: Google 공식 쿼터 문서 기준 다수 확인)*
- **핵심:** 우리 서비스는 **플랫폼이 유저 영상을 대신 업로드·검색하지 않는다.** 레시피는 유저가 **자기 계정·자기 Google Cloud 프로젝트에서** 실행하므로, 10,000유닛 쿼터는 우리가 아니라 각 유저가 각자 보유한다. 즉 이 API의 쿼터 한계가 우리 플랫폼의 확장을 막지 않는다.
- **제약 인지:** 레시피 안내 시 "업로드는 하루 약 100건 한도" 같은 쿼터 정보를 유저에게 명시해, 유저가 자기 한도 안에서 쓰도록 가이드한다.

### 실행 가능성 요약
핵심 기능(F2·F4)은 Claude API 호출만으로 구현되며, 비용·기술 난이도 모두 팀 부트캠프 범위 안에 있다. 무거운 외부 데이터 파이프라인이나 대량 API 쿼터가 필요 없는 구조라, **MVP 기간 내 실제 동작하는 서비스 출시가 현실적으로 가능하다.**

## 5. 핵심 성과지표 (KPI)

'얼마나 봤는가'가 아니라 **'얼마나 실제로 완성했는가'** 를 측정한다.

- ★ **레시피 복사 수** — 실제로 가져다 썼는가
- ★ **변형 실행 수** — 내 상황에 맞게 바꿨는가
- ★ **에러 해결 전환율** — 막힌 뒤 끝까지 완성했는가

보조 지표: 재방문율, 공유 링크 경유 유입, SNS 도달.

이 데이터는 가상 매출이 아니라 **실제 유저 행동 로그**로, 팀 전원의 포트폴리오 근거가 된다.
